# 4_1_事件與事故管理
```
事件:發生時有造成極大的損失時，就變成了資安事故。

事故:防毒軟體偵測到惡意程式、系統發生警訊等等

網路瞬斷一秒，被系統紀錄到，或許使用者們皆無感覺，這就像是事件。若網路斷線一小時，惹發民怨、要求賠償、影響商譽這就算是事故。(嚴重性)
```

# 4_2_備援與營運持續
```
異地備援（英語：remote backup）指的是在另一地點備份與建立重要的資料、伺服器以及工作機能等，避免因火災、水災、遭竊等災難造成重要資料損失與工作、服務中斷。其可分為冷備援模式、暖備援模式、熱備援模式以及傳統自建備援模式
https://zh.wikipedia.org/wiki/%E7%95%B0%E5%9C%B0%E5%82%99%E6%8F%B4
```
```
營運持續管理（Business Continuity Management, BCM）
https://phoenixnap.com/blog/what-is-business-continuity-management
業務連續性管理被定義為組織的高級計劃和準備，以維持業務功能或在災難發生後快速恢復。它還涉及定義潛在的風險，包括火災，洪水或網絡攻擊。
商業領袖計劃在危機發生之前識別並解決潛在危機。然後測試這些過程以確保它們有效，並定期檢查該過程以確保它是最新的。

```
